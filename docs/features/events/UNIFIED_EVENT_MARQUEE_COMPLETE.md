### ✅ 修改后：统一的跑马灯UI

```swift
// 统一的赛事活动跑马灯通知（附近活动 + 同城活动）
let eventsToShow: [EventService.Event] = {
    if let nearbyEvent = eventManager.nearbyEvent {
        // 优先显示附近活动
        return [nearbyEvent.event]
    } else {
        // 否则显示同城活动（50km内）
        return eventManager.localCityEvents.filter {
            $0.status == "published" || $0.status == "active"
        }
    }
}()

if !eventsToShow.isEmpty {
    EventMarqueeNotification(events: eventsToShow)
        .padding(.top, 8)   // 统一位置
        .zIndex(150)        // 统一层级
}
```

**优势**：
- ✅ UI完全统一（同一个组件）
- ✅ 位置一致（统一在top: 8）
- ✅ 所有活动都有跑马灯效果
- ✅ 代码简洁清晰
- ✅ 优先级明确：附近活动 > 同城活动

## 🎯 显示逻辑

### 优先级规则

```
1. nearbyEvent 存在？
   ├─ Yes → 显示该附近活动（单个）
   └─ No  → 显示所有同城活动（50km内，多个轮播）

2. 状态过滤
   └─ 只显示 "published" 或 "active" 状态的活动
```

## 🔧 技术实现细节

### 事件优先级算法

```swift
let eventsToShow: [EventService.Event] = {
    // Step 1: 检查是否有附近活动（用户在活动区域内）
    if let nearbyEvent = eventManager.nearbyEvent {
        // 返回单个附近活动（数组形式，兼容跑马灯组件）
        return [nearbyEvent.event]
    }

    // Step 2: 返回同城活动（50km内）
    else {
        // 过滤：只要 published 或 active 状态
        return eventManager.localCityEvents.filter {
            $0.status == "published" || $0.status == "active"
        }
    }
}()
```

### 跑马灯适配

EventMarqueeNotification自动适配单个或多个活动：

- **单个活动**：不显示轮播指示器，不自动切换
- **多个活动**：显示指示器，每4秒切换
