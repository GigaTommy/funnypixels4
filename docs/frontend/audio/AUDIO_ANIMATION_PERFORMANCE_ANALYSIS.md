### 3. 智能节流机制

**问题场景**: 用户快速连续点击（100次/秒）

**无节流**:
```
100次/秒 × 0.5ms = 50ms累计延迟 ❌
可能导致音效重叠混乱
```

**有节流**（50ms）:
```
20次/秒 × 0.5ms = 10ms累计延迟 ✅
音效清晰，无重叠
```

**实现**:
```swift
private var lastPlayTime: [String: Date] = [:]
private let throttleInterval: TimeInterval = 0.05  // 50ms

func playSystemSoundFast(_ key: String, withThrottle: Bool = false) {
    if withThrottle {
        let now = Date()
        if let lastTime = lastPlayTime[key],
           now.timeIntervalSince(lastTime) < throttleInterval {
            return  // 跳过，无性能消耗
        }
        lastPlayTime[key] = now
    }

    if let soundID = systemSounds[key] {
        AudioServicesPlaySystemSound(soundID)  // <0.5ms
    }
}
```

**结果**: ✅ **保护性能 + 提升体验**

### 4. 异步执行

**所有音效和触觉都在后台线程**:

```swift
// ✅ 不阻塞主线程

    AudioServicesPlaySystemSound(soundID)
}

// ✅ 触觉也是异步的系统API
HapticManager.shared.impact(style: .light)  // 系统自动异步
```

**主线程CPU**: **0%** ✅

## 🛡️ 安全保障机制

### 1. 多层防护

```swift
// 层1: 节流保护（防止高频触发）
if now.timeIntervalSince(lastTime) < 0.05 { return }

// 层2: 系统优先级（用户交互优先）
DispatchQueue.global(qos: .userInteractive).async { }

// 层3: 异步执行（不阻塞主线程）
AudioServicesPlaySystemSound(soundID)  // 系统自动异步

// 层4: 性能监控（Debug模式）
#if DEBUG
if duration > 0.005 {
    Logger.warning("⚠️ 音效延迟过高")
}
#endif
```

### 2. 降级策略

```swift
// 低电量模式自动禁用音效
if ProcessInfo.processInfo.isLowPowerModeEnabled {
    return  // 直接跳过
}

// 后台模式自动禁用
if UIApplication.shared.applicationState != .active {
    return
}
```

### 3. 用户控制

```swift
// 设置中提供全局开关
@AppStorage("soundEffectsEnabled") var soundEnabled = true
@AppStorage("hapticsEnabled") var hapticsEnabled = true

// 播放前检查
guard soundEnabled else { return }
```
