# 3D像素城市 - 任务优先级总结

**更新时间**: 2026-03-10
**基于**: 14点核心产品需求分析

---

## 🎯 执行策略

### 核心原则

> **"如果用户进入 3D 城市第一秒不会说 '哇'，那这个系统就是失败的。"**

**三个必须解决的产品问题**:
1. ✅ 用户看到自己的贡献 → **Find my pixel** (#60)
2. ✅ 用户看到城市在成长 → **视觉震撼** (#61, #64)
3. ✅ 用户愿意再次参与建设 → **建造动画** (#65)

---

## 📋 当前任务状态

### ✅ 已完成（技术基础）
- #54: GPU Instancing实现 ✅
- 基础3D渲染、LOD系统、相机控制 ✅
- 点击检测、详情加载 ✅

### ❌ 严重缺失（产品核心）
- **产品定位**: Map/City未分离（用户会迷失）
- **核心体验**: 缺少"Find My Pixel"（最重要功能）
- **第一印象**: 无震撼感（无灯光动画、无bloom）
- **视觉规则**: 高度未对数压缩（可能失衡）

---

## 🔥 Phase 0: 架构基础（必须先做）

### 为什么必须先做？
当前架构不支持产品核心理念：
- TowerSceneView = 地图透明叠加层
- 用户无法区分"绘制"和"观看"
- 违反需求二："3D城市不是地图增强功能，而是独立模式"

### 任务清单

| 任务ID | 任务名称 | 优先级 | 预估 | 阻塞项 |
|-------|---------|--------|------|--------|
| **#58** | **Map/City独立模式架构重构** | 🔴 P0 | 3-4天 | **阻塞所有Phase 1任务** |
| **#59** | **对数高度压缩算法** | 🔴 P0 | 0.5天 | 无 |
| #68 | 城市数据模型整理 | 🟡 P1 | 1天 | 无 |

**Phase 0 总计**: 4.5-5.5天

**关键产出**:
- [x] ViewMode枚举 (.map / .city)
- [x] CityView独立组件
- [x] 模式切换UI和动画
- [x] 对数高度算法 (height = 20*log10(pixelCount) + 5)

---

## ⭐ Phase 1: 核心产品体验（最高优先级）

### 为什么是最高优先级？
这些功能直接决定产品成败：
- **#60**: "Find My Pixel" - 被强调3次的最重要功能
- **#61**: 城市进入动画 - 决定第一印象的"哇"
- **#62**: 极简UI - 产品体验基础

### 任务清单

| 任务ID | 任务名称 | 优先级 | 预估 | 产品价值 |
|-------|---------|--------|------|----------|
| **#60** | **Find My Pixel功能** | 🔴 P0 | 2-3天 | ⭐⭐⭐⭐⭐ (最高) |
| **#61** | **城市进入动画（第一印象）** | 🔴 P0 | 2天 | ⭐⭐⭐⭐⭐ |
| **#62** | **城市UI极简化** | 🟡 P1 | 1天 | ⭐⭐⭐⭐ |

**Phase 1 总计**: 5-6天

**关键产出**:
- [x] "Find My Pixel"金色按钮（右下角）
- [x] 镜头飞行到用户像素（3秒动画）
- [x] Cube金色高亮（脉冲闪烁）
- [x] 提示气泡："你在第2412层！"
- [x] 城市进入7秒动画序列（太阳升起→建筑亮起→相机拉远）
- [x] 极简UI（屏幕90%是3D场景）

**验收标准**:
```
用户测试：
1. 第一次进入City → 说"哇！"
2. 点击"Find My Pixel" → 3秒飞到目标 → 金色高亮 → 有成就感
3. UI不遮挡视野 → 清楚知道在"看"不是在"画"
```

## 📈 成功指标

### Phase 1完成后必须达到：

**定性指标**:
- [ ] 5/5用户第一次进入说"哇"或类似惊叹词
- [ ] 5/5用户能找到"Find My Pixel"按钮
- [ ] 5/5用户点击后表达成就感（笑、点头、评论）
- [ ] 5/5用户清楚知道在"看城市"不是"画地图"

**定量指标**:
- [ ] 城市进入动画<8秒
- [ ] Find My Pixel飞行<4秒
- [ ] FPS ≥ 30
- [ ] 内存 < 800MB

### 最终成功标准

> **"用户会开始说：我在广州塔第2412层"**

当用户自发产生这种内容时，产品即成功。
