# GPS位置偏移问题修复

## 🐛 问题描述

**现象**：
- 用户在TestLocationPicker中选择"张民达墓"
- GPS绘制时地图自动跳转到"广州动物园"
- 最终绘制的像素偏离起点100-200米

## 🔍 根本原因

### 修复前：累积偏移（随机游走）

```swift
var currentCoord = baseCoord  // 起点
for i in 1...10 {
    let offset = random(10-20米)
    currentCoord = currentCoord + offset  // ⚠️ 累积偏移
    绘制(currentCoord)
}
```

**轨迹示意**:
```
张民达墓 (起点)
    ↓ +15米
  点1
    ↓ +12米
  点2
    ↓ +18米
  点3
    ↓ +20米
  ...
    ↓
  点10 → 广州动物园 (偏移了150米！)
```

**问题**：
- 每次都在**上一个点**的基础上累积偏移
- 10次绘制后，累积偏移可达**100-200米**
- 地图GPS跟随会跟着"漂移"，最终到达很远的地方

### 修复后：辐射分布

```swift
let baseCoord = testCenter  // 起点固定
for i in 1...10 {
    let offset = random(5-15米)
    let coord = baseCoord + offset  // ✅ 从起点辐射
    绘制(coord)
}
```

**轨迹示意**:
```
       点3      点5
         \      /
    点2 - 张民达墓 - 点7
         /  ⭐  \
       点1      点9
         \      /
        点4    点10
```

**改进**：
- 所有点都围绕**起点**（张民达墓）分布
- 最大偏移：**5-15米**（固定半径）
- 地图始终停留在选择的位置附近

## ✅ 修复详情

**文件**: `FunnyPixelsApp/Views/ContentView.swift`

**关键改动**:
```swift
// 修复前：
currentCoord = CLLocationCoordinate2D(
    latitude: currentCoord.latitude + deltaLat,   // 累积
    longitude: currentCoord.longitude + deltaLng  // 累积
)

// 修复后：
let currentCoord = CLLocationCoordinate2D(
    latitude: baseCoord.latitude + deltaLat,   // 从起点计算
    longitude: baseCoord.longitude + deltaLng  // 从起点计算
)
```

**参数优化**:
```swift
// 修复前：
let distanceMeters = Double.random(in: 10...20)  // 10-20米

// 修复后：
let distanceMeters = Double.random(in: 5...15)   // 5-15米（更紧凑）
```

## 📝 技术说明

### 为什么使用辐射分布？

1. **符合GPS测试场景**:
   - 模拟用户在一个固定地点周围移动
   - 类似于"在咖啡馆内移动"、"在公园散步"等场景

2. **更好的测试覆盖**:
   - 测试同一网格内的多次绘制
   - 测试相邻网格的渲染
   - 避免测试范围过大导致无法验证

3. **用户体验**:
   - 地图不会突然跳转
   - 绘制结果可预测
   - 符合"在这里绘制"的直觉

### 未来改进

可以考虑添加两种模式：

**模式1：固定点绘制**（当前实现）
- 所有点在起点周围小范围分布
- 适合测试特定位置

**模式2：轨迹绘制**（未来功能）
- 模拟真实的移动轨迹
- 沿着道路或路径绘制
- 适合测试长距离绘制
