📝迁移与种子文件生成规范

1️⃣ 迁移文件生成规范

文件命名规则：

格式为 YYYYMMDD_NNN_description.js

例如：

20250912_001_create_users.js

20250912_002_create_pixels.js

20250912_003_add_pixels_user_fk.js

顺序控制：

迁移文件必须严格按时间戳顺序执行。

外键依赖必须单独写在后续迁移文件中，避免依赖顺序错误。

迁移粒度：

一个迁移文件只做一类变更（建表 / 加字段 / 加外键 / 改索引）。

不要在同一个文件里混合多个表的改动。

安全变更：

新增字段 → 默认允许 nullable 或提供默认值。

删除字段 → 先标记弃用，再在下一个版本里 drop。

索引变更 → 先新增，再删除旧的。

回滚对称性：

up 和 down 方法必须对称，确保迁移失败后可回滚。

2️⃣ 种子文件生成规范

环境区分：

seeds/dev/* → 开发环境假数据。

seeds/test/* → 测试环境数据，可重复执行。

seeds/prod/* → 生产环境初始化必要配置（如管理员账号、基础参数），避免插入测试数据。

幂等设计：

使用 upsert 或 on conflict do nothing，保证多次执行不会产生重复或脏数据。

命名规则：

格式为 YYYYMMDD_description.js

例如：

20250912_init_admin_user.js

20250912_insert_default_patterns.js

3️⃣ 开发流程要求

所有迁移和种子文件在提交前必须：

本地完整跑一次 migrate:latest → seed:run → rollback。

确认能从空数据库跑到最新状态。

4️⃣ 角色定位

你（AI助手）只负责生成迁移和种子文件的 草稿。

文件生成后，开发者会负责：

审核命名和顺序。

确认依赖是否正确。

再合并到主分支。
