**创建时间**: 2026-02-23
**状态**: ✅ 已完成
**问题**: 用户在活动区域内看不到活动入口

---

## 🎯 问题描述

用户反馈：
> "我当前定位在区庄地铁站，为何收不到活动推送，地图页看不到活动入口？"

**根本原因**:
1. ❌ 地图页没有检测附近活动的功能
2. ❌ 没有显示活动区域范围的可视化
3. ❌ 用户进入活动区域时没有任何提示

---

## ✅ 解决方案


#### 2. 附近活动横幅 (NearbyEventBanner)

**显示位置**: 地图顶部中央
**显示时机**: 当检测到2公里内有活动时
**交互**: 点击横幅可导航到活动详情页

**显示内容**:
- ✅ 活动图标（绿色=在区域内，蓝色=在区域外）
- ✅ 活动名称
- ✅ 距离信息
  - 在区域内："您在活动区域内！"
  - 在区域外："距离 XXX米/公里"
- ✅ 活动状态标签（PUBLISHED/ACTIVE）
- ✅ 点击箭头提示

**视觉效果**:
- 圆形图标 + 渐变背景
- 阴影效果增强层次感
- 动画过渡（滑入/滑出）

#### 3. 自动位置监听

**集成点**: `EventManager.setupLocationObservation()`

```swift
LocationManager.shared.$currentLocation
    .sink { location in
        self?.checkGeofence(location: location.coordinate)
        self?.checkNearbyEvents(location: location.coordinate) // ⭐️ 新增
    }
```

每次GPS位置更新时自动检测附近活动。

## 🔧 技术细节

### 距离计算

使用 CoreLocation 的精确距离计算：
```swift
let eventLocation = CLLocation(latitude: center.lat, longitude: center.lng)
let userLocation = CLLocation(latitude: location.latitude, longitude: location.longitude)
let distance = userLocation.distance(from: eventLocation) // 米
```

### 性能优化

1. **检测范围限制**: 只检测2公里内的活动
2. **状态过滤**: 只检测 `published` 和 `active` 状态
3. **实时更新**: 跟随GPS位置自动更新，无需手动刷新
4. **单例模式**: EventManager 单例，避免重复计算

### 线程安全

所有UI更新都在主线程：
```swift
DispatchQueue.main.async {
    self.nearbyEvent = closestEvent
}
```
