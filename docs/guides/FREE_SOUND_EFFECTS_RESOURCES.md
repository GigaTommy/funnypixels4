## 📥 下载与使用流程

### 步骤 1: 访问推荐平台
1. **优先访问 Pixabay** (最方便，无需注册，CC0 授权)
2. **备选 Mixkit** (需注册，但授权清晰)
3. **高级需求用 Freesound** (最大库，但需筛选授权)

### 步骤 2: 搜索音效
- 使用提供的**搜索关键词**
- 试听多个音效，选择最符合场景的
- 检查**时长** (过长音效需要剪辑)

### 步骤 3: 检查授权
确认音效授权类型:
- ✅ **CC0**: 完全免费，无需署名
- ✅ **Pixabay License**: 免费商用，无需署名
- ✅ **Mixkit License**: 免费商用，无需署名
- ⚠️ **CC-BY**: 免费商用，**需在 App 设置页署名**
- ⚠️ **Zapsplat 免费版**: 需在项目中署名 (升级 Pro 版无需)

### 步骤 4: 下载音效
- 选择 **WAV** 格式 (无损，方便后期处理)
- 或 **MP3** 格式 (体积小，直接使用)
- 建议下载 **44.1kHz 或 48kHz 采样率**

### 步骤 5: 音效处理 (可选)
使用免费工具处理音效:

**推荐工具**:
- **Audacity** (免费开源): 剪辑、淡入淡出、降噪
- **GarageBand** (macOS): 专业音频编辑
- **在线工具**: [Audio Trimmer](https://audiotrimmer.com/), [MP3Cut](https://mp3cut.net/)

**常见处理**:
1. **剪辑**: 裁剪到需要的时长
2. **淡入淡出**: 避免音效突兀开始/结束
3. **音量标准化**: 统一所有音效音量 (-3dB 至 -6dB 峰值)
4. **格式转换**: 转换为 iOS 推荐的 **CAF** 或 **M4A** 格式

### 步骤 6: 导入 Xcode
```bash
# 1. 将音效文件放入项目
/FunnyPixelsApp/FunnyPixelsApp/Resources/Sounds/

# 2. 拖拽到 Xcode 项目
确保勾选: ✅ Copy items if needed
             ✅ Add to targets: FunnyPixelsApp

# 3. 在 Build Phases > Copy Bundle Resources 中验证
```

### 步骤 7: 更新代码
参考 `AUDIO_ANIMATION_UX_ENHANCEMENT_PLAN.md` 中的代码实现

## 🔧 音效优化技巧

### 格式选择

| 格式 | 优点 | 缺点 | iOS 推荐 |
|------|------|------|---------|
| **WAV** | 无损，高质量 | 文件大 | ⚠️ 需压缩 |
| **MP3** | 体积小，通用 | 有损压缩 | ✅ 可用 |
| **M4A (AAC)** | 高质量，小体积 | 需转换 | ⭐ 最佳 |
| **CAF** | iOS 原生 | 仅 Apple 平台 | ⭐ 最佳 |

### 推荐配置
```
格式: M4A (AAC)
比特率: 128 kbps (足够音效使用)
采样率: 44.1 kHz
声道: 单声道 (UI 音效不需要立体声)
```

### 转换命令 (使用 FFmpeg)
```bash
# 安装 FFmpeg
brew install ffmpeg

# 批量转换 WAV/MP3 到 M4A
for file in *.wav; do
  ffmpeg -i "$file" -c:a aac -b:a 128k -ar 44100 -ac 1 "${file%.wav}.m4a"
done

# 或转换为 CAF (iOS 原生)
afconvert -f caff -d aac -b 128000 input.wav output.caf
```

### 音量标准化
确保所有音效音量一致:
```bash
# 使用 FFmpeg 标准化音量到 -3dB
ffmpeg -i input.m4a -af "volume=-3dB" output.m4a

# 或使用 Audacity 的 "Normalize" 效果 (图形界面)
```
