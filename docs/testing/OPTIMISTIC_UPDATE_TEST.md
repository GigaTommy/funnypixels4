# 🚀 乐观更新测试指南

## 功能说明

实现了**乐观更新（Optimistic Update）**机制，用户绘制完成后**立即**看到今日任务进度更新，无需等待后端处理。

---

## 实现原理

### 旧流程（慢）：
```
用户结束会话
  ↓
调用后端 API (等待2-5秒)
  ↓ 批处理刷新、计算统计、更新任务
后端返回成功
  ↓
发送刷新通知
  ↓
再次调用 API 获取任务列表
  ↓
UI 更新 ⏱️ 总耗时: 3-7秒
```

### 新流程（快）：
```
用户结束会话
  ↓
立即本地更新 UI ⚡ <-- 用户立即看到
  ↓ (0ms延迟)
异步调用后端 API
  ↓
后端返回后同步数据 ✓
```

---

## 测试步骤

### 测试1: 正常场景 - 乐观更新成功

1. **前置条件**：
   - 今日任务中有 `draw_sessions` 和 `draw_pixels` 任务
   - 当前进度例如：`draw_sessions: 0/3`, `draw_pixels: 0/50`

2. **操作**：
   - 开始GPS绘制
   - 绘制 5 个像素
   - 点击"结束"按钮

3. **预期结果**：
   - ⚡ **立即**（0延迟）看到任务进度更新：
     - `draw_sessions: 0/3` → `1/3` ✅
     - `draw_pixels: 0/50` → `5/50` ✅
   - 界面显示震动反馈
   - 约2秒后后端同步完成（后台）

4. **日志验证**：
```
🚀 已发送乐观更新通知: sessions=1, pixels=5
📊 本地更新任务进度: sessions=1, pixels=5
  ✅ draw_sessions: 0 → 1/3
  ✅ draw_pixels: 0 → 5/50
  📊 任务进度: 0/5
✅ Backend session ended: xxx-xxx-xxx
✅ 已发送后端同步通知
```

## 技术细节

### 通知 userInfo 结构：

**乐观更新：**
```swift
[
  "optimisticUpdate": true,
  "sessionCompleted": 1,
  "pixelsDrawn": 5
]
```

**后端同步：**
```swift
[
  "optimisticUpdate": false
]
```

**回滚：**
```swift
[
  "optimisticUpdate": false,
  "rollback": true
]
```

## 注意事项

1. **乐观更新只适用于本地可预测的操作**
   - ✅ 会话计数（+1）
   - ✅ 像素计数（已知数量）
   - ❌ 地图任务（需要后端计算位置）

2. **后端失败时会自动回滚**
   - 网络错误
   - 服务器错误
   - 验证失败

3. **后端成功后仍会同步数据**
   - 确保与服务器状态一致
   - 修正任何计算偏差
