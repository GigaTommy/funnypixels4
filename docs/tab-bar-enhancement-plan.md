# FunnyPixels 底部菜单栏功能完善方案

> 基于地图社交运动游戏的定位，从游戏产品运营角度出发，针对五个 Tab（地图、历史、联盟、排行榜、我的）进行系统性功能升级规划。
> 目标：提升信息密度、增强用户参与度、提高用户粘性、构建完整的游戏社交体验闭环。

## 1. 地图 Tab

### 1.1 当前状态分析

已有功能：MVT 像素地图渲染、手动/GPS 绘画、像素点击查看详情、漫游热点、战争区域、漂流瓶、成就提示、每日签到。

**核心问题**：地图作为主战场，缺乏「社交可见性」和「实时对抗感」，用户在地图上感受不到其他玩家的存在和竞争。

### 1.2 功能完善方案

#### A. 实时社交存在感系统

| 功能 | 描述 | 交互逻辑 | 目的 |
|------|------|----------|------|
| **附近玩家雷达** | 地图上以脉冲光点显示正在绘画的附近玩家（模糊位置，保护隐私） | 点击光点 → 弹出简要卡片（头像、昵称、联盟旗帜、正在绘画的颜色） → 可关注/打招呼/查看主页 | 让用户感知到「这个世界有其他人在玩」 |
| **实时绘画轨迹流** | 当缩放级别 ≥14 时，以半透明动画显示最近 5 分钟内他人的绘画轨迹 | 轨迹渐隐动画，类似流星效果 → 点击可查看绘画者信息 | 增强地图活力感 |
| **区域人气热力图** | 地图底层叠加一层区域活跃度热力图（可开关） | Toolbar 新增热力图按钮 → 显示近 24h 各区域活跃度 → 颜色从冷到热 | 引导用户前往热门区域参与竞争 |
| **好友在线状态** | 已关注的好友在地图上显示专属标记（星形/心形图标） | 好友标记自动出现 → 点击可发起快速互动（点赞、喊话、导航前往） | 增强社交连接 |

#### B. 领地控制与竞争可视化

| 功能 | 描述 | 交互逻辑 | 目的 |
|------|------|----------|------|
| **联盟领地边界** | 在 zoom 10-14 显示联盟领地的边界线和颜色填充 | 联盟占领超过某区域 60% 像素后自动形成领地 → 边界高亮 → 点击显示领地信息（占领联盟、像素数、控制时长） | 增强领地归属感和竞争意识 |
| **争夺热区标记** | 多个联盟交叉渗透的区域显示「争夺中」特殊标记 | 闪烁的交叉旗帜图标 → 点击查看参战联盟和各自占比 → 可一键参战 | 刺激用户参与领地争夺 |
| **领地变更通知** | 用户所在联盟的领地被侵入时推送地图内通知 | 地图顶部弹出警报条 → 显示「XX联盟正在入侵XX区域」→ 点击飞往该区域 | 激发防守行为和即时参与 |
| **个人领地高亮** | 长按地图空白处 → 切换为「我的领地」视图，高亮显示自己绘画过的所有像素 | 高亮模式下其他像素灰化 → 显示个人覆盖统计（总面积、城市分布） | 增强成就感和归属感 |

#### C. 地图内任务与挑战系统

| 功能 | 描述 | 交互逻辑 | 目的 |
|------|------|----------|------|
| **每日地图任务** | 地图上随机生成 3-5 个任务点位（带图标标记） | 标记闪烁引导 → 点击查看任务详情（如「在此处绘画 10 个像素」「连续绘画 500 米」）→ 完成后弹出奖励动画 | 日活引导和目标驱动 |
| **限时区域挑战** | 每天定时开放特定区域的限时挑战（如「30 分钟内在XX公园绘画最多像素的人获胜」） | 地图上出现倒计时区域 → 进入后自动加入挑战 → 顶部显示实时排名 → 结束弹出结果 | 制造紧迫感和竞技氛围 |
| **宝箱/资源点** | 地图上随机生成可拾取的资源点（积分宝箱、限时道具、稀有颜色） | 靠近资源点 50 米内 → 地图上图标变大+弹跳 → 点击拾取 → 开箱动画 | 激励用户出门探索 |
| **路线挑战** | 系统发布特定路线挑战（如沿某条街完整绘画） | 地图上显示虚线引导路线 → GPS 绘画沿路线行进 → 偏离路线则中断 → 完成整条路线获得额外奖励 | 鼓励用户探索新路线 |

#### D. 快捷信息面板

| 功能 | 描述 | 交互逻辑 | 目的 |
|------|------|----------|------|
| **地图顶部状态条** | 显示当前区域名称、区域内像素总数、活跃玩家数、天气/时段主题 | 始终显示在地图顶部 → 下拉展开详细区域信息 → 包含区域排行快照 | 增强信息密度 |
| **快速统计浮窗** | 右侧 Toolbar 新增个人统计按钮 | 点击弹出半透明浮窗 → 显示今日像素数、连续登录天数、当前排名、积分余额 | 无需切换 Tab 即可查看核心数据 |
| **地图内快捷聊天** | 地图左下角的气泡按钮 | 点击展开附近/联盟聊天面板 → 半屏显示 → 可发送预设快捷语和表情 → 支持位置分享 | 增强即时社交互动 |

#### E. 地图主题与氛围

| 功能 | 描述 | 目的 |
|------|------|------|
| **昼夜主题切换** | 根据用户当地时间自动切换地图明暗风格（日间/夜间/黄昏/黎明） | 增强沉浸感 |
| **节日/赛季主题** | 在节日或赛季期间更换地图底图风格（如春节红色主题、万圣节暗黑主题） | 运营事件驱动 |
| **天气效果叠加** | 根据当地实际天气在地图上叠加粒子效果（雨滴、雪花、阳光光晕） | 增强真实感和趣味性 |

## 2. 历史 Tab

### 2.1 当前状态分析

已有功能：绘画记录网格/列表展示、日期/城市筛选、Session 详情（地图轨迹+统计数据）、轨迹回放、分享卡片。

**核心问题**：纯粹的「个人记录查看器」，缺乏社交维度和数据洞察，信息密度极低，无法引导用户回访。

### 2.2 功能完善方案 — 重新定位为「动态中心」

> **关键策略：将"历史"Tab 升级为"动态"Tab，融合个人记录 + 社交 Feed + 数据洞察。**

#### A. 顶部 Sub-Tab 重构

将原有的单一记录列表拆分为三个子标签：

| Sub-Tab | 名称 | 内容 | 目的 |
|---------|------|------|------|
| **动态** | 社交 Feed | 关注的人的绘画动态、联盟动态、系统公告 | 社交驱动，参考微信朋友圈/微博 |
| **我的** | 个人记录 | 即原有绘画历史（保留网格/列表切换、筛选功能） | 保留原有功能 |
| **数据** | 数据仪表盘 | 个人绘画数据的可视化分析 | 增强数据洞察和成就感 |

#### B. 社交动态 Feed（新增）

| 功能 | 描述 | 交互逻辑 | 目的 |
|------|------|----------|------|
| **关注动态流** | 展示已关注用户的最新绘画 Session、成就达成、排名变动 | 时间线排列 → 每条动态含用户头像+昵称+时间 → 内容卡片（轨迹缩略图+统计摘要）→ 互动按钮（赞、评论、分享、飞往该位置） | 构建社交信息流 |
| **联盟动态** | 联盟内成员的活动汇总（新成员加入、领地变更、成就达成、战争结果） | 联盟 Logo 前缀 → 事件描述 → 可点击相关人/地点 | 增强联盟归属感 |
| **附近动态** | 当前位置附近 5km 内的公开绘画动态 | 按距离排序 → 显示距离标签 → 点击飞往地图查看 | 发现附近玩家 |
| **点赞与评论** | 每条动态支持点赞、简短评论 | 点赞有动画效果 → 评论区折叠式展开 → 支持 @提及 → 回复通知 | 社交互动闭环 |
| **动态发布** | 用户可手动发布「圈子」动态（图文+位置标签） | 发布按钮（右上角 +）→ 选择图片/截图 → 添加文字和位置标签 → 选择可见范围 | 用户自主内容生产 |

#### C. 数据仪表盘（新增）

| 功能 | 描述 | 展示形式 | 目的 |
|------|------|----------|------|
| **周/月报告** | 自动生成绘画数据报告（类似微信读书周报） | 精美卡片式报告 → 包含本周像素数、排名变化、最活跃时段、最常去的区域 → 可分享 | 定期复访驱动 |
| **趋势图表** | 像素数量、活跃天数、运动距离的时间趋势图 | 折线图/柱状图 → 支持 7天/30天/90天/全部 时间范围切换 | 数据可视化 |
| **城市足迹地图** | 小型地图展示用户去过的所有城市，点亮式效果 | 中国/世界地图 → 已到访城市点亮 → 显示城市总数/占比 → 点击城市查看该城市的绘画统计 | 收集驱动、分享驱动 |
| **运动健康统计** | 关联 HealthKit，展示通过绘画产生的步数、卡路里、运动时长 | 环形进度条 → 日目标/周目标 → 与 Apple Health 数据对比 | 健康运动激励 |
| **对比面板** | 与上周/上月自己的数据对比 + 与好友数据对比 | 双柱对比图 → 绿色↑红色↓箭头标识涨跌 → 好友对比需双方同意 | 竞争驱动 |
| **绘画热力日历** | 类 GitHub Contribution Graph 的日历热力图 | 每日格子颜色深浅代表活跃度 → 连续活跃天数高亮 → 点击某天查看当天详情 | 连续登录激励 |

#### D. 个人记录增强

| 功能 | 描述 | 交互逻辑 | 目的 |
|------|------|----------|------|
| **收藏标记** | 允许用户收藏特定 Session | 长按 Session 卡片 → 添加到收藏 → 收藏筛选器 | 个人精选管理 |
| **标签分类** | 用户可为 Session 添加自定义标签（如「跑步」「通勤」「探索」） | Session 详情页添加标签按钮 → 输入或选择已有标签 → 按标签筛选 | 个性化记录管理 |
| **Session 对比** | 选择两个 Session 进行数据对比 | 多选模式 → 选择两条记录 → 并排展示轨迹和数据对比 | 自我进步追踪 |
| **年度回顾** | 年末自动生成年度绘画回顾报告 | 精美动画展示 → 总像素/总距离/总时长/去过的城市/最佳 Session → 可分享为长图或视频 | 年度留存 + 传播 |
| **绘画作品集** | 以「作品集」视角展示连续绘画形成的图案/路线艺术 | 瀑布流展示 → AI 自动识别有意义的图案 → 用户可编辑标题和描述 | 增强成就感 |

## 3. 联盟 Tab

### 3.1 当前状态分析

已有功能：我的联盟列表、联盟搜索、联盟详情、成员管理（角色/踢人/转让）、联盟创建/编辑/解散、申请审核、邀请系统、旗帜选择。

**核心问题**：联盟仅作为「组织管理工具」存在，缺乏联盟内社交互动、共同目标、成长体系、实时沟通，联盟的存在感和凝聚力极弱。

### 3.2 功能完善方案

#### A. 联盟主页重构

将联盟详情页升级为信息丰富的「联盟主页」：

| 模块 | 内容 | 展示形式 | 目的 |
|------|------|----------|------|
| **联盟概况 Header** | 联盟旗帜（大图）、名称、等级、成员数/上限、总像素数、排名、联盟宣言 | Hero banner 样式 → 旗帜作为背景 → 核心数据横排展示 | 视觉冲击 + 快速信息获取 |
| **联盟公告板** | 盟主/管理员发布的公告，置顶显示 | 顶部滚动公告 → 点击展开全文 → 支持富文本和图片 | 组织通信 |
| **联盟领地概览** | 小地图显示联盟领地分布 | 缩略地图 + 领地色块 → 点击跳转到地图 Tab 对应位置 | 领地可视化 |
| **近期战报** | 联盟参与的最近战争/事件结果 | 时间线列表 → 胜/负/平标识 → 点击查看详细战报 | 战争信息传达 |

#### B. 联盟等级与成长系统（新增）

| 功能 | 描述 | 规则逻辑 | 目的 |
|------|------|----------|------|
| **联盟等级** | 1-50 级，根据经验值升级 | 经验来源：成员绘画像素（每像素 1 点）、完成联盟任务（50-500 点）、赢得战争（1000-5000 点）、成员日活跃（每活跃成员每天 10 点） | 长期成长目标 |
| **等级特权** | 每个等级解锁新功能/权限 | 5 级解锁联盟聊天室、10 级成员上限+10、15 级解锁联盟任务板、20 级解锁联盟商店、25 级解锁领地防御工事、30 级解锁联盟战争发起权 | 升级动力 |
| **联盟贡献值** | 每位成员的贡献度量化展示 | 贡献 = 绘画像素 × 1 + 任务完成 × 10 + 签到 × 5 + 捐赠积分 × 1 → 周/月贡献排行 → 贡献决定分红比例 | 激励个人贡献 |
| **联盟资金/国库** | 成员可向联盟捐赠积分，形成联盟公共资金 | 捐赠入口 → 资金用于：升级联盟、发起战争、购买联盟道具、发放成员奖励 → 消费记录透明可查 | 经济系统闭环 |

#### C. 联盟任务系统（新增）

| 功能 | 描述 | 交互逻辑 | 目的 |
|------|------|----------|------|
| **每日联盟任务** | 系统每日发布 3-5 个联盟协作任务 | 任务板展示 → 示例任务：「联盟成员今日总计绘画 1000 像素」「3 名成员在同一区域绘画」「联盟在线人数同时达到 5 人」→ 完成后所有在线成员获得奖励 | 鼓励联盟协作 |
| **周常挑战** | 每周一个大型挑战目标 | 进度条展示 → 全联盟协力完成 → 多档奖励（完成 50%/80%/100% 分别获得不同奖励）→ 完成后联盟获得经验 | 中期协作目标 |
| **联盟战役** | 盟主可发起对特定区域的攻占战役 | 盟主选择目标区域 → 设定时间范围（1-7 天）→ 联盟成员在该区域绘画贡献进度 → 战役结束结算 | 战略玩法 |
| **联盟接力** | 系统生成一条长路线，需要联盟成员接力绘画完成 | 路线地图展示 → 每位成员只能绘画一段 → 接力棒图标标记当前进度 → 全部完成获得联盟成就 | 团队协作乐趣 |

#### D. 联盟社交功能（新增）

| 功能 | 描述 | 交互逻辑 | 目的 |
|------|------|----------|------|
| **联盟聊天室** | 实时文字聊天（基于现有 Socket.IO） | 联盟主页内嵌聊天入口 → 消息列表 → 支持文字/表情/位置分享/快捷指令 → 未读消息 badge | 实时团队沟通 |
| **战术地图白板** | 盟主/管理员可在地图上标注战术点位 | 编辑模式 → 在地图上圈选区域、标注攻击/防守点 → 标注自动同步给所有成员 → 成员地图上可见 | 战略协调 |
| **成员动态流** | 联盟内成员的最新活动汇总 | 时间线：XX 完成了一次绘画（+120 像素）/ XX 达成了成就 / XX 升级了 → 可互动（点赞/评论） | 联盟内社交粘性 |
| **联盟签到** | 每日联盟签到，与个人签到独立 | 签到按钮 → 签到可获得联盟贡献值 → 展示今日已签到成员头像列表 → 全员签到额外奖励 | 联盟日活驱动 |

#### E. 联盟外交系统（新增）

| 功能 | 描述 | 交互逻辑 | 目的 |
|------|------|----------|------|
| **联盟关系** | 设定与其他联盟的外交关系：友好/中立/敌对 | 联盟设置中选择目标联盟 → 发送外交请求 → 对方盟主确认 → 友好联盟在地图上用绿色标记、敌对用红色 | 增加社交复杂度和策略性 |
| **联盟联赛** | 多个联盟之间的定期联赛/锦标赛 | 系统每月组织 → 报名 → 分组 → 小组赛（比谁在限定区域绘画多）→ 淘汰赛 → 决赛 → 颁奖 | 竞技内容 |
| **联盟合并/分裂** | 支持两个联盟合并或联盟分裂 | 合并需双方盟主同意 → 保留高等级联盟的名称和等级 → 分裂需盟主操作 | 联盟生态管理 |

## 4. 排行榜 Tab

### 4.1 当前状态分析

已有功能：个人/联盟/城市三维度排行、日/周/月/全部时间段切换、点赞功能。

**核心问题**：纯静态列表展示，无竞技驱动、无段位体系、无社交激励、无奖励机制，看完排名无后续行为指引。

### 4.2 功能完善方案

#### A. 排行榜维度扩展

在现有 个人/联盟/城市 基础上增加以下子维度：

| 新增维度 | 描述 | 排序指标 | 目的 |
|---------|------|----------|------|
| **好友排行** | 仅展示已关注/互关用户的排名 | 像素数、运动距离、活跃天数 | 小圈子竞争更有代入感 |
| **附近排行** | 基于用户当前位置，展示 5km/10km/50km 范围的排行 | 像素数 | 本地竞争刺激 |

## 5. 我的 Tab

### 5.1 当前状态分析

已有功能：个人资料卡（头像/昵称/签名/积分）、荣誉展示、菜单入口（商店/活动中心/消息/背包/成就/旅途收藏/邀请好友/设置）、登出。

**核心问题**：信息展示过于简陋，更像一个设置入口页面而非「个人中心」，缺乏社交属性、进度追踪、个性化展示。

### 5.2 功能完善方案

#### A. 个人主页全面重构

| 模块 | 内容 | 展示形式 | 目的 |
|------|------|----------|------|
| **Profile Hero** | 头像（支持自定义边框）、昵称、段位徽章、称号（可选一个佩戴）、签名、所属联盟 | 全宽 Hero Banner → 背景可自定义（解锁或购买） → 头像右下角联盟旗帜小徽章 → 段位图标显眼位置 | 视觉丰富、身份展示 |
| **核心数据横栏** | 关注数、粉丝数、总像素、总成就 | 四栏横排 → 每栏可点击进入对应详情 → 关注/粉丝进入社交列表页 | 关键数据一目了然 |
| **今日卡片** | 今日像素数/目标完成度、连续登录天数、每日任务进度 | 渐变色卡片 → 环形进度条 → 快捷入口按钮（开始绘画、今日任务） | 日活引导 |
| **段位进度** | 当前段位 → 下一段位的进度条 | 段位图标 + 经验条 → 显示还差多少像素升级 → 预计天数 | 成长目标感 |

#### B. 社交关系系统（新增/增强）

| 功能 | 描述 | 交互逻辑 | 目的 |
|------|------|----------|------|

## 7. 跨 Tab 联动机制

### 7.1 核心联动路径

```
地图 Tab (核心玩法)
  ↕ 绘画产生数据
历史 Tab (数据沉淀)
  ↕ 分享产生社交
联盟 Tab (组织协作)
  ↕ 竞争产生排名
排行榜 Tab (竞争激励)
  ↕ 成就驱动成长
我的 Tab (个人中心)
  ↕ 任务引导回地图
地图 Tab (循环闭环)
```

### 7.2 关键联动设计

| 联动场景 | 涉及 Tab | 交互流 | 目的 |
|---------|---------|--------|------|
| **绘画完成 → 查看动态** | 地图 → 历史 | Session 结束 → Summary 弹窗含「查看详情」按钮 → 跳转历史 Tab 的 Session 详情 | 自然行为引导 |
| **动态互动 → 前往地图** | 历史 → 地图 | 社交 Feed 中看到好友的绘画 → 点击「飞往」→ 跳转地图 Tab 并飞往该位置 | 社交驱动地图探索 |
| **联盟任务 → 地图执行** | 联盟 → 地图 | 联盟任务板中接受任务 → 点击「前往」→ 跳转地图 Tab 并标记任务区域 | 任务驱动行为 |
| **排名变化 → 立即行动** | 排行榜 → 地图 | 看到自己被超越 → 点击「去绘画」→ 跳转地图 Tab → 开始绘画 | 竞争驱动行为 |
| **每日任务 → 各处执行** | 我的 → 各 Tab | 任务列表中的每个任务带有快捷入口按钮 → 一键跳转到对应 Tab | 任务引导全功能 |
| **成就解锁 → 全局通知** | 任意 → Toast | 任何 Tab 中达成成就 → 顶部弹出成就 Toast → 点击跳转成就详情 | 成就感即时反馈 |
